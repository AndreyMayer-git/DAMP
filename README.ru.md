## Обзор

DAMP записывает и анализирует *динамику* вынужденного выбора между кратко
предъявленными эмоциональными изображениями. Вместо одного лишь финального выбора он
фиксирует, как разворачивается решение — моторную конкуренцию (траекторию мыши) и
стабильность выбора — в виде индексов уровня процесса. Только для исследований; без
клинической или диагностической интерпретации.

## Дизайн эксперимента

**Стимулы.** Изображения из базы OASIS (Kurdi et al., 2017), классифицированные по
валентности (**Позитив / Негатив / Нейтрал**), затем обработанные: удаление фона,
перевод в оттенки серого, гауссово размытие и заливка одним плоским цветом. **Цвет
назначается независимо от валентности** (скрещён по категориям), поэтому оттенок и
яркость не могут служить прокси аффективной категории. Обработанный набор отбирается
вручную — оценщиком(ами), отличными от автора стимулов и, в идеале, вслепую к
валентности: изображение оставляют, только если объект остаётся различимым и
занимает ≳ 40% полотна.

**Ход пробы.** (1) звуковой сигнал → (2) фиксационный крест (2.0–3.5 с) →
(3) два стимула предъявляются последовательно (тайминг ниже) → (4) окно ответа 3.0 с
на пустом экране → (5) подтверждение + межпробный интервал (1.0–2.5 с). Маскировка не
используется.

**Тайминг** (джиттер на каждой пробе против предугадывания):

| Параметр | Диапазон |
|---|---|
| Экспозиция стимула (каждого) | 18–30 мс (3–5 кадров @ 165 Гц) |
| Межстимульный интервал | 80–150 мс |

**Типы и состав проб:**

| Тип | Кол-во | |
|---|---|---|
| Позитив–Нейтрал | 17 | с нейтралом |
| Негатив–Нейтрал | 17 | с нейтралом |
| Позитив–Негатив | 17 | оба валентны |
| SWAP (зеркальное повторение) | 10 | проверка стабильности |
| **Всего** | **61** | |

> **О ярлыках.** «оба валентны» против «с нейтралом» задаёт контраст DCI по *составу стимулов*. Действительно ли условие с обоими валентными стимулами даёт большее отклонение траектории (DCI > 0) — это открытый эмпирический вопрос ниже, а не допущение (препринт §2.2 / §4.2).

Пропущенные пробы (нет ответа за 3.0 с) повторяются в конце и исключаются из основного
анализа.

## Калибровка

7 проб с кликом по квадрату, появляющемуся случайно слева или справа; медиана RT
становится индивидуальной базой (`base_rt`) для нормировки RT.

## Запись

- Траектория мыши на **100 Гц** (каждые 10 мс) в течение всей пробы
- Время реакции (от начала окна ответа до клика)
- Цветовая категория каждого стимула (описательный анализ смещений)

## Поведенческие индексы

**Основные индексы:**

| Индекс | Формула | Интерпретация |
|---|---|---|
| **DCI** — индекс конфликта решения | `(AUC_high − AUC_low) / AUC_low` | Относительное отклонение траектории, оба валентны vs с нейтралом. Выполняется ли DCI > 0 — **открытый эмпирический вопрос**, здесь не доказано (см. препринт §2.2 / §4.2). Для группового вывода разностный контраст `AUC_high − AUC_low` устойчивее этого отношения. |
| **RSI** — индекс стабильности репрезентации | `N_same / N_swap` | Доля SWAP-проб, где исходный выбор сохраняется несмотря на пространственную инверсию. Выше уровня случайности ⇒ стабильность предпочтения (оценивать через доверительный интервал). |

AUC = интеграл перпендикулярного отклонения от идеальной прямой (метод трапеций).
В DCI `AUC_high` — это условие с обоими валентными стимулами (Позитив–Негатив), а `AUC_low` — пары с Нейтралом; «высокий/низкий» обозначают состав стимулов, а не предполагаемую величину конфликта.

**Исследовательский индекс:**

| Индекс | Определение | Примечание |
|---|---|---|
| **FLIP** | Число пересечений траекторией центральной вертикальной оси (x = 0), усреднённое по пробам | Записывается, но **исключён из основного аналитического фреймворка** — низкая теоретическая специфичность, чувствителен к моторному шуму (препринт §4.1). |

## Вычислительная модель и симуляции

В препринте **нет данных о людях**; все рисунки модели — симуляция из генеративной
модели Раздела 2.2. Две конкурирующие фальсифицируемые модели задают предсказанный
**знак DCI**:

- **Модель 1 — минимальный drift-diffusion** (`dampsimulation.py`): предсказывает
  **DCI ≤ 0** (наибольший разрыв валентности, Позитив–Негатив, разрешается быстрее →
  самый прямой путь). Устойчива к параметру шума σ.
- **Модель 2 — leaky competing accumulator** (`damp_lca_model.py`; Usher & McClelland,
  2001): предсказывает **DCI > 0** при весе захвата значимости γ > 0.

Поэтому знак DCI — эмпирический вопрос для будущих данных, а не результат симуляции.

## Использование

1. Поместите исходные изображения OASIS в `Database_2026/images/`, сгруппированные
   по валентности.
2. Запустите `Database_2026/script.py` для стандартизации (удаление фона, ч/б,
   блюр, плоский цвет). Результат пишется в `Database_2026/processed_stimuli/`.
3. Вручную отберите стимулы из `processed_stimuli/`: оставляйте изображение, только
   если объект по-прежнему узнаваем и занимает ≳ 40% полотна; остальное удаляйте.
   Отбор должны проводить оценщик(и), отличные от автора стимулов, в идеале вслепую
   к валентности. Перенесите оставленные изображения в
   `Database_2026/final_standardized_stimuli/` с подпапками `Positive/`,
   `Negative/`, `Neutral/` (≥ 50 изображений в каждой).
4. Запустите `main.py` — введите ID участника, возраст, пол; пройдите согласие
   и калибровку; затем основная задача (~40 мин).
5. Запустите `analyze_data.py`, чтобы построить индексы и рисунки для всех сессий в
   `data/`.

**Пилотный режим** (`use_fixed_stimuli = True`): одинаковые стимулы и тайминг для всех
участников через `pilot_template.json` (создаётся автоматически, если его нет).
**Случайный режим** (`= False`): полная рандомизация на каждую сессию.
